# Prescio Twitter Strategy Guide

## 핵심 철학: "Receipts-First Build in Public"

증거 기반 빌딩. 말보다 결과로 보여주기.

---

## 트윗 빈도

- **목표:** 일 3-5회 (주 20-25회)
- **시간대:** 
  - 아침 9-10시 (아시아)
  - 저녁 9-10시 (미국 저녁, 아시아 아침)

---

## 콘텐츠 믹스

| 유형 | 비율 | 설명 |
|------|------|------|
| 진행 업데이트 | 40% | Shipping receipts, 개발 현황 |
| 교육/인사이트 | 25% | AI 에이전트, 예측 마켓 인사이트 |
| 커뮤니티 인터랙션 | 20% | 질문, 피드백 요청, RT |
| 밈/유머 | 15% | 캐주얼한 콘텐츠, 실패담 |

---

## 트윗 포맷

### 1. Shipping Receipt (주 2-3회)
```
📊 Day X — Shipping Receipt

✅ 완료:
- [구체적 기능/작업]
- [측정 가능한 결과]

📈 현황:
- 트레저리: $X
- 게임 수: X개
- 참여자: X명

🔗 receipts: [onchain link]

#BuildInPublic #Prescio
```

### 2. 진행 업데이트 (일상)
```
🔮 [오늘 작업한 것]

[1-2문장 설명]

[스크린샷 or GIF 첨부]
```

### 3. 질문/피드백 요청
```
Quick question for the community:

[구체적 질문]

Drop your thoughts 👇
```

### 4. 마일스톤 발표
```
🚀 MILESTONE

[달성한 것] 

Thank you to everyone who [기여한 방식]

Next up: [다음 목표]

[이미지/스크린샷]
```

### 5. 캐주얼/실패담
```
[실패한 것]. oh well.

[배운 점 or 다음 시도]

[유머러스한 마무리]
```

## 브랜딩 요소

### 이모지
- 🔮 예측/미래
- 📊 데이터/통계
- 🚢 shipping
- 🎭 Among Us/게임
- 🤖 AI 에이전트

### 해시태그
- #BuildInPublic (항상)
- #Prescio (브랜드)
- #Monad (생태계)
- #AIAgents (주제)

### 톤
- **캐주얼하지만 전문적**
- 실패도 솔직하게 공유
- 숫자와 증거로 뒷받침
- 커뮤니티에 감사 표현

## 반드시 지킬 것 (Non-negotiables)

1. **Show proof every time** - 스크린샷, 온체인 링크
2. **Ship small, often** - 작은 것도 공유
3. **Ask for specific action** - 단순 발표 X, CTA 포함
4. **Tie product → token → fees** - 토큰 유틸리티 연결

---

## 피해야 할 것

- ❌ 빈 약속, 로드맵만 공유

- ❌ 커뮤니티 피드백 무시
- ❌ 너무 형식적인 톤
