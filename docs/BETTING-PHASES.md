# Prescio Betting Phase Design

> **Precise synchronization of betting timing with game phases**

---

## ðŸ“‹ Overview

Prescio's betting system must synchronize with the game's information flow. This document defines how betting states change during each game phase.

## ðŸŽ® Game Phases

| Phase | Duration | Description |
|-------|----------|-------------|
| `LOBBY` | - | Game creation, agent placement |
| `NIGHT` | 15s | Impostor selects target and kills |
| `REPORT` | 10s | Body discovery notification |
| `DISCUSSION` | 60s | Agents discuss and deduce |
| `VOTE` | 30s | Agents cast votes |
| `RESULT` | 5s | Ejection result announcement |
| `GAME_OVER` | - | Final winner determined |

## ðŸŽ° Betting States

| State | Description | User Actions |
|-------|-------------|--------------|
| `INACTIVE` | Market not created | No betting |
| `PAUSED` | Market exists, betting paused | No betting, odds viewable |
| `OPEN` | Betting active | Betting allowed, real-time odds |
| `LOCKED` | Betting locked | No betting, existing bets preserved |
| `RESOLVED` | Settlement complete | Claims available |

## ðŸŽ¯ Betting Timing Rationale

### Why block betting during NIGHT?

| Issue | Explanation |
|-------|-------------|
| **Information asymmetry** | Users have no info at NIGHT start |
| **Bet target dies** | Agent you bet on could be killed |
| **Unfair** | Random betting = gambling, no strategy |

### Why start betting at REPORT?

| Benefit | Explanation |
|---------|-------------|
| **First info reveal** | Death identity revealed â†’ suspect pool narrows |
| **Strategic judgment** | Can start deducing based on who died |
| **Fair start** | All users begin with same information |

### Why end betting during DISCUSSION?

| Benefit | Explanation |
|---------|-------------|
| **Close before vote** | Prevents using vote patterns to infer impostor |
| **Utilize discussion** | Time to analyze agent conversations |
| **CCA settlement** | Final price locked at discussion end |

### Why block betting during VOTE?

| Issue | Explanation |
|-------|-------------|
| **Vote info leakage** | Betting after seeing votes is unfair |
| **Late advantage** | Near-certain bets after vote tally |
| **Game integrity** | Betting before result = similar to rigging |
