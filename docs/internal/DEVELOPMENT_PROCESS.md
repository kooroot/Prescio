# Prescio 개발 프로세스

## 역할 분담

| 역할 | 담당 |
|------|------|
| **prescio-pm** | 기획, 기능명세서 작성 |
| **coder** | 구현, 코드 작성 |
| **auditor** | 코드 오딧, 보안 검토 |
| **prescio-worker** | 운영, 배포, 모니터링 |

## 개발 플로우

```
┌─────────────────────────────────────────────────────────────┐
│  1. PM: 기획/기능명세서 작성                                  │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  2. Coder + Auditor + Worker: 검토/피드백                    │
│     - 고도화 또는 단순화 제안                                 │
│     - 기술적 feasibility 확인                                │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 확정 → Coder: 구현                                       │
│     - feature branch 생성 (feat/*, fix/*)                    │
│     - 단계별 커밋                                            │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  4. Auditor: 오딧/검토/피드백                                 │
│     - 보안 취약점 체크                                       │
│     - 코드 품질 검토                                         │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  5. 크리티컬 이슈 있으면 → 재개발 (3번으로)                    │
│     없으면 → PR 생성                                         │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  6. GitHub: PR 생성 + 리뷰 + Merge to main                   │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  7. Worker: 운영/배포                                        │
└─────────────────────────────────────────────────────────────┘
```

## Branch 전략

- `main`: 프로덕션
- `feat/*`: 새 기능
- `fix/*`: 버그 수정

## PR 규칙

- 모든 기능/단계별 커밋
- Auditor 리뷰 필수
- 크리티컬 이슈 없을 시 머지
